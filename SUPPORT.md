# Support Guide

WildScape Europe is maintained through a lightweight GitHub-first support model. The repository documentation is designed to answer setup, validation, release, and deployment questions before an issue is opened.

## Before opening an issue

| Question | Start here |
|---|---|
| How do I run the app locally? | `docs/GETTING_STARTED.md` |
| How is the architecture organized? | `docs/ARCHITECTURE.md` |
| How do I test changes? | `docs/TESTING.md` |
| How do I deploy Release 1.0? | `docs/DEPLOYMENT.md` |
| What is the repo hygiene standard? | `docs/REPO_HYGIENE.md` |

## Issue quality

A useful support request includes the command that failed, the complete error message, the operating system, the Node.js version, and the commit SHA. If the issue is a behavior bug, include the expected result, the actual result, and the smallest set of steps that reproduces the problem.

## Maintainer triage

| Priority | Criteria |
|---|---|
| High | Build, validation, security, or release blockers on `main`. |
| Medium | Incorrect app behavior with a clear reproduction path. |
| Low | Documentation refinements, visual polish, or non-blocking enhancement requests. |
